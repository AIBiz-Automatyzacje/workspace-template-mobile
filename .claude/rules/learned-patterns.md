# Learned Patterns

Reguły wyciągnięte z rozwiązanych problemów w docs/solutions/. Zarządzane przez /dev-compound i /dev-compound-refresh.

<!-- rule-count: 2 -->

- **E2E Maestro nie steruje natywnymi powierzchniami OS — wstrzykuj dane przez service_role, nie tapuj pickera**: Maestro steruje wyłącznie a11y tree aplikacji RN. Natywnych powierzchni systemu — picker zdjęć, kamera, picker dokumentów, share sheet, natywny `Alert`/`ActionSheet` (confirm/destructive), biometria — NIE dotknie; flow oparty o nie wisi na timeout, a `[E2E]` cicho degraduje do OPERATOR. Wzorzec autora scenariusza: prześledź flow krok po kroku; jeśli krok wymaga natywnej powierzchni OS → (a) dane, które wpadłyby przez picker/kamerę, wstrzyknij przez service_role (`runScript: .maestro/inject-*.js`, wzór `.maestro/etap-12-inject-message.js` — REST insert kluczem service_role, omija RLS i natywne UI; trigger realtime = realny dowód toru danych) i asertuj RENDER (siatka/miniatura/teaser); (b) natywny confirm/destructive → `Operator checklist` jako `[Manual]`. Nigdy nie pisz flow tapującego natywny picker. Pełna tabela granic: skill `mobile-e2e-maestro` §„Granica możliwości Maestro". Reguła egzekwowana też w `dev-plan` §3.4b i `dev-docs` (self-check `[E2E]`).
  Source: etap-12b galeria mediów (2026-06-22) — plan miał flow tapujący picker iOS, ręczna korekta na inject; wzór: etap-12 czat realtime.

- **Skrypt inject (`runScript`) leci w GraalJS Maestro, nie w Node — binarkę przekazuj inline, nie ścieżką pliku**: Skrypty `runScript: .maestro/inject-*.js` wykonują się w silniku GraalJS Maestro, NIE w Node.js. Brak dostępu do filesystemu (`http` w GraalJS nie czyta plików z dysku), brak `require`/modułów. Upload binarki (np. obraz placeholder do Storage) z body `{filePath: '...'}` cicho NIE działa — body musi być inline string/base64. W etap-12b ten jeden szczegół zablokował WSZYSTKIE 4 flow E2E i wyszedł dopiero przy realnym `maestro test` (autopilot napisał skrypt, ale nigdy go nie uruchomił). Reguła: skrypt inject = czysty REST (`http`) + inline payload; sekrety przez `env:` w YAML flow (nigdy w repo); wzór 1:1 `.maestro/etap-12-inject-message.js`. Meta-lekcja: deliverable E2E napisany, lecz nieuruchomiony = bug ukryty do pierwszego realnego przebiegu — „green" bez wykonania jest fałszywy.
  Source: etap-12b galeria mediów (2026-06-23) — realny run 4/4 PASS po fixie body uploadu z {filePath} na string.
